<p>Millions of users work with SQL to keep the gears of their business turning. In an era marked by relentless digital transformation, the proliferation of AI workloads, and tightening regulatory demands, the journey of these organizations is uniquely their own. At Microsoft, we believe beneath this diversity lies a single, unified, and consistent backbone: one consistent SQL.</p>



<p><strong>One consistent SQL is a promise of a unified experience </strong>that spans edge, cloud, and software as a service (SaaS), empowering developers to create intelligent applications with ease, while enabling data professionals to modernize on their terms. Write once, deploy anywhere because customers&nbsp;run diverse SQL estates, and we meet them wherever they are. Whether you’re running workloads with SQL Server, modernizing in <a href="https://azure.microsoft.com/en-us/products/azure-sql" rel="noreferrer noopener" target="_blank">Azure SQL</a>, or activating real-time analytics in <a href="https://www.microsoft.com/en-us/microsoft-fabric" rel="noreferrer noopener" target="_blank">Microsoft Fabric</a>, <strong>one consistent SQL unifies your data estate</strong>, bringing <strong>platform consistency, performance at scale, advanced security, and AI-ready tools</strong> together in one seamless experience and creates <strong>one home for your SQL workloads in the era of AI</strong>.</p>



<p>With a <strong>consistent platform, optimized performance, </strong>and <strong>multi-layered security</strong>, workloads scale dynamically while staying secure and highly available. And building intelligent, scalable applications is faster than ever with developer-first tools combined with AI capabilities. This is enterprise-grade performance for the era of AI, with one consistent SQL powering innovation across hybrid, multi-cloud, and AI-powered environments, all anchored by a <strong>unified data estate</strong>.</p>



<h2 class="wp-block-heading" id="delivering-a-consistent-sql-platform-across-cloud-and-on-premises">Delivering a consistent SQL platform across cloud and on-premises</h2>



<p>From edge to cloud, one consistent SQL is Microsoft’s commitment to deliver a consistent experience—powering mission-critical workloads, next-generation apps, and data-driven intelligence. It provides unified management across infrastructure, platform, and software services for consistent operations in hybrid scenarios. Developers can use familiar SQL skills and tools to build and manage applications across environments, while sovereign cloud support addresses data residency and compliance requirements for regulated industries.</p>



<p>And now, the centerpiece of this vision takes the stage: <strong><a href="https://aka.ms/sqlserver2025blog" rel="noreferrer noopener" target="_blank">SQL Server 2025 is now generally available</a></strong>.<strong> </strong>Built on SQL Server’s foundation of security, performance, and availability, this release introduces built-in AI capabilities and developer-focused enhancements. These features enable organizations to use existing data to support AI initiatives securely and at scale, all within SQL Server using the T SQL language.</p>

<h2 class="wp-block-heading" id="achieving-optimized-sql-performance-and-advanced-security">Achieving optimized SQL performance and advanced security</h2>



<p>In a world where trust is non-negotiable, organizations expect the same performance, availability, and security standards everywhere in their data lives. They adapt to the demands of your workload without compromise—delivering resilience and speed while keeping data secure and operations steady. These expectations are heightened for mission-critical workloads, where uptime and reliability directly impact return on investment. Businesses need a platform that not only meets these requirements but also evolves to handle growing complexity without adding operational burden.</p>



<p><strong>The next generation of Azure SQL Managed Instance is now&nbsp;</strong><a href="https://aka.ms/mi-next-gen-gp-blog" rel="noreferrer noopener" target="_blank"><strong>generally available</strong></a>&nbsp;to help organizations modernize with improved performance and simplified migration.&nbsp;The release offers expanded storage and database capacity, flexible&nbsp;compute&nbsp;and memory options, and features designed to support diverse workload requirements.&nbsp;These enhancements provide a foundation for scaling applications and managing data securely while&nbsp;maintaining&nbsp;compatibility with existing SQL investments.&nbsp;Customers are already benefiting from Azure SQL Managed Instance. <a href="https://www.microsoft.com/en/customers/story/25083-hexure-azure-sql-managed-instance?msockid=0b58992db71663d937eb8b1eb6f66291#customers-share-modal-dialog" rel="noreferrer noopener" target="_blank">Hexure</a>, a life insurance software company, slashed processing time by 97.2%.</p>



<p>The path to Azure SQL Managed Instance also got easier.&nbsp;<strong>SQL Server migration in Azure Arc</strong> is now <a href="https://aka.ms/arc-migrations-ga-blogpost" rel="noreferrer noopener" target="_blank">generally available</a>. Copilot-assisted migration streamlines the entire process with real-time replication, confident cutover, and trusted failover—reducing months of work to days and lowering total cost of ownership.</p>

<h2 class="wp-block-heading" id="empowering-developers-with-ai-ready-sql-tools-at-scale">Empowering developers with AI-ready SQL tools at scale</h2>



<p>One consistent SQL means you can use your existing skills and familiar development tools together with AI-powered capabilities to simplify application creation. As organizations look to integrate generative AI into business processes, complexity and time to value remain major challenges. Analysts predict that most AI applications will be built on existing data platforms—reducing development effort and accelerating delivery. Customers want integration, not disruption, and they expect AI assistance embedded in the tools they already use.</p>



<p>This vision is supported by features that combine intelligence with security.&nbsp;GitHub&nbsp;Copilot integration enables developers to work with AI&nbsp;assistance&nbsp;directly in environments like <a href="https://azure.microsoft.com/en-us/products/visual-studio-code" rel="noreferrer noopener" target="_blank">Visual Studio Code</a> and SQL Server Management Studio 22.&nbsp;Native support for&nbsp;retrieval of&nbsp;augmented generation scenarios includes vector search and semantic indexing&nbsp;within&nbsp;SQL Server and&nbsp;Azure SQL, while secure enclaves and Always Encrypted protect sensitive data during processing.&nbsp;Connections to <a href="https://azure.microsoft.com/en-us/products/ai-services" rel="noreferrer noopener" target="_blank">Azure AI services</a> and governance tools streamline data preparation and compliance, while elastic infrastructure supports training, inference, and deployment at scale.&nbsp;</p>



<h2 class="wp-block-heading" id="creating-a-unified-data-estate-for-analytics-and-ai">Creating a unified data estate for analytics and AI</h2>



<p>Microsoft Fabric connects operations, analytics, and governance in one unified experience, unlocking the full potential of SQL data.&nbsp;As organizations&nbsp;prepare for&nbsp;real-time AI applications, the ability to bridge transactional and analytical workloads becomes essential.&nbsp;Analysts describe this capability as “translytical,” a foundation for responsive applications that combine speed with depth.&nbsp;A future-ready data strategy must integrate operational data with analytical insight, and Fabric delivers that integration without adding complexity.aka</p>



<p>Today at Microsoft Ignite 2025, we announced two key advancements:&nbsp;<strong>the general availability of Fabric databases and <a href="https://learn.microsoft.com/en-us/fabric/mirroring/overview" rel="noreferrer noopener" target="_blank">database Mirroring in Fabric</a></strong>.&nbsp;Fabric databases unify operational and analytical data within Fabric, backed by&nbsp;<a href="https://learn.microsoft.com/en-us/fabric/onelake/onelake-overview" rel="noreferrer noopener" target="_blank">OneLake</a>, the open data foundation for analytics and AI. <a href="https://learn.microsoft.com/en-us/fabric/database/sql/overview" rel="noreferrer noopener" target="_blank">SQL database in Fabric</a> supports transactional processing, real-time analytics&nbsp;with zero Extract, Transform, and Load (ETL), and AI workloads side-by-side.&nbsp;In addition, database Mirroring enables replication of SQL Server&nbsp;and&nbsp;Azure SQL&nbsp;into Fabric for analytics and AI scenarios without migration or refactoring.&nbsp;Early adopters such as&nbsp;<a href="https://www.microsoft.com/en/customers/story/25614-ap-pension-microsoft-fabric" rel="noreferrer noopener" target="_blank">AP Pension</a>&nbsp;have&nbsp;consolidated&nbsp;decades of&nbsp;fragmented data&nbsp;using&nbsp;Mirroring for SQL Server in Fabric and SQL database in&nbsp;Fabric, creating a centralized architecture with automated delivery and strong governance.&nbsp;These capabilities position Fabric as a cornerstone for organizations&nbsp;seeking&nbsp;to modernize data strategies and prepare for AI-powered applications.</p>



<p><a href="https://aka.ms/Eastman" rel="noreferrer noopener" target="_blank">Eastman</a> built a new agentic sales copilot app, enabling their sellers to instantly query unified customer data using natural language. By integrating SQL querying with Fabric’s analytics and vector capabilities, the copilot retrieves hyper-precise answers directly from the database.</p>
