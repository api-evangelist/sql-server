<p>We take pride in delivering innovation with each new version of Microsoft SQL Server. However, there comes a time when product lifecycles must conclude. On July 14, 2026, SQL Server 2016 will reach its 10-year end-of-support moment. Many of our customers, including <a href="https://www.microsoft.com/en/customers/story/23849-national-yunlin-university-of-science-and-technology-azure" rel="noreferrer noopener" target="_blank">YunTech</a>, have begun transitioning their SQL workloads to Microsoft Azure or are upgrading to SQL Server 2025. Their objective is straightforward: to modernize their databases and applications while accelerating innovation through using cloud technologies. For customers who need more time, Microsoft will offer three years of Extended Security Updates for SQL Server 2016.</p>



<blockquote class="wp-block-quote blockquote is-layout-flow wp-block-quote blockquote-is-layout-flow">
<p>&#8220;We are allowing the cloud provider to handle hardware resource allocation and maintenance so that our staff focus on program development. This strategy ensures that during system operation we no longer need to worry about hardware failures, power instability or information security issues, greatly improving the system&#8217;s operational reliability.&#8221;</p>
<cite><em>—Ching-Lung Chang, CIO, Library and Information Services Office at YunTech </em>.</cite></blockquote>



<h2 class="wp-block-heading" id="modernize-to-azure-a-smooth-path-a-more-powerful-destination">Modernize to Azure, a smooth path, a more powerful destination &nbsp;</h2>



<p>Migrating to a cloud platform is an essential step on the journey to modernization, and there are many choices. What makes SQL Server and Microsoft Azure SQL unique is that it’s built on the same engine, no matter where you deploy, which means you can build on your existing SQL experience while gaining the layered security, intelligent threat detection, and data encryption that Azure provides. &nbsp;</p>



<p>Modernizing to Microsoft Azure SQL Managed Instance offers cost savings, scalability, security, seamless migration, productivity, and always up-to-date features. Now in preview, Azure SQL Managed Instance next generation general purpose delivers improved performance and scalability, making migration and modernization faster and easier across more customer scenarios. &nbsp;</p>

<p>Azure is the destination, but we know the journey matters just as much. A new SQL Server <a href="https://www.microsoft.com/en-us/sql-server/blog/2025/07/17/accelerate-sql-server-migration-to-azure-with-azure-arc/" rel="noreferrer noopener" target="_blank">migration experience</a> is now under preview in Azure Arc. It is powered by Azure Database Migration Service and offers seamless, end-to-end migration capabilities including continuous migration assessments, simplified provisioning, and real-time database replication, assisted by Copilot in Azure. What once took months can now be accomplished in just days, with confidence, continuity, and control.</p>



<h2 class="wp-block-heading" id="in-place-upgrade-to-sql-server-2025">In-place upgrade to SQL Server 2025 &nbsp;</h2>



<p>Another way to stay protected is to upgrade your SQL Server to  <a href="https://www.microsoft.com/en-us/sql-server/blog/2025/05/19/announcing-sql-server-2025-preview-the-ai-ready-enterprise-database-from-ground-to-cloud/" rel="noreferrer noopener" target="_blank">SQL Server 2025</a>. Built on SQL Server&#8217;s legacy of best-in-class security, performance and availability, SQL Server 2025 empowers you to develop modern AI applications using your data. It provides built-in, extensible AI capabilities, enhanced developer productivity, and seamless integration with Azure and Fabric, all within SQL Server engine using the familiar T-SQL language.&nbsp;</p>



<p>The upgrade experience has been streamlined. With the retirement of Azure Data Studio and Data Migration Assistant, migration capabilities are now integrated directly into SQL Server Management Studio (SSMS). This eliminates the need for separate tools, reducing complexity and effort. In SSMS 21, a new migration extension allows DBAs and partners to assess and upgrade SQL Server instances from older to newer versions, all within the same management environment.&nbsp;</p>

<h2 class="wp-block-heading" id="frequently-asked-questions">Frequently asked questions&nbsp;</h2>



<h3 class="wp-block-heading" id="what-does-end-of-support-mean">What does end of support mean? </h3>



<p>Microsoft Lifecycle Policy offers 10 years of support (five years for mainstream support and five years for extended support) for business and developer products (such as SQL Server and Windows Server). After the end of the extended support period, there are no patches or security updates, which might cause security and compliance issues, and expose your applications and business to serious security risks.&nbsp;</p>



<h3 class="wp-block-heading" id="what-do-extended-security-updates-include">What do Extended Security Updates include?&nbsp;</h3>



<p>Extended Security Updates include provision of security updates and bulletins rated <strong>critical</strong> by the <a href="https://msrc.microsoft.com/update-guide" rel="noreferrer noopener" target="_blank">Microsoft Security Response Center</a> (MSRC), for a maximum of three years after the end of extended support.&nbsp;&nbsp;</p>



<p>Extended Security Updates are distributed if and when available. Extended Security Updates don&#8217;t include technical support. Customers must purchase a paid support plan (Pay Per Incident, Unified, and Premier Support for Partners) to leverage technical support. Extended Security Updates don&#8217;t include new features, functional improvements, nor customer-requested fixes. However, Microsoft might include non-security fixes as deemed necessary.&nbsp;</p>



<h3 class="wp-block-heading" id="why-do-extended-security-updates-only-offer-critical-updates">Why do Extended Security Updates only offer &#8220;critical&#8221; updates?&nbsp;</h3>



<p>For end of support events in the past, SQL Server provided only critical security updates, which meets the compliance criteria of our enterprise customers. SQL Server doesn&#8217;t ship a general monthly security update. </p>
